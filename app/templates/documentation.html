{{ _('docs.section.use') }}

{% if locale == 'es' %}

Listados (/): tabla de anuncios deduplicados por URL canónica. Puedes filtrar para ver solo los resultados de la última ejecución correcta o ver el histórico completo. El botón Ejecutar ahora lanza un scraping manual (puede tardar varios minutos; el botón se desactiva al enviar).

Desde cada fila puedes abrir el anuncio en el portal y enviar la URL a la Blacklist para ocultarla en el listado y en futuros merges.

Idioma y tema: el selector de idioma guarda preferencia en una cookie; el tema claro/oscuro se guarda en localStorage. La versión mostrada en la cabecera corresponde a APP_VERSION en el despliegue.

{% elif locale == 'en' %}

Listings (/): table of deduplicated listings by canonical URL. Filter to show only rows from the last successful run or view the full history. Run now starts a manual scrape (can take several minutes; the button disables on submit).

From each row you can open the listing and send its URL to the Blacklist to hide it from the table and future merges.

Language and theme: language is stored in a cookie; light/dark theme is stored in localStorage. The header version pill reflects APP_VERSION in the deployment.

{% else %}

Inserate (/): Tabelle deduplizierter Inserate nach kanonischer URL. Filter für nur die letzte erfolgreiche Ausführung oder die komplette Historie. Jetzt ausführen startet einen manuellen Scrape (kann mehrere Minuten dauern; Button wird beim Absenden deaktiviert).

Pro Zeile: Inserat im Portal öffnen oder URL zur Blacklist senden (Ausblendung und keine zukünftigen Merges).

Sprache und Design: Sprache per Cookie; helles/dunkles Theme in localStorage. Versions-Badge = APP_VERSION des Deployments.

{% endif %}

{{ _('docs.section.config') }}

{% if locale == 'es' %}

En Config (/config) se guardan ajustes en la base de datos SQLite:

  • Intervalo del scheduler (horas): frecuencia del scraping automático tras reiniciar la aplicación.
  • Proveedor LLM: OpenAI-compatible (remoto) u Ollama (local, API /v1).
  • Base URL, API key, modelo y temperature para las llamadas al LLM.
  • Estrategia de listado (HTML → candidatos) y un único prompt LLM (mensaje de sistema) para la extracción JSON de listados.

La aplicación resuelve sola la URL efectiva del endpoint (prioridad: variable de entorno GPM_LLM_BASE_URL → valor guardado → por defecto la API de OpenAI). En entornos locales prueba varias direcciones hasta que GET /v1/models responda. Tras guardar, la caché de resolución se invalida. El enlace Probar abre /api/llm-check en otra pestaña con un informe JSON.

{% elif locale == 'en' %}

Settings (/config) persist values in SQLite:

  • Scheduler interval (hours): automatic scrape cadence after an app restart.
  • LLM provider: OpenAI-compatible (remote) or Ollama (local /v1 API).
  • Base URL, API key, model, and temperature.
  • Listing strategy (HTML → candidates) and a single LLM prompt (system message) for JSON listing extraction.

The app auto-resolves the effective base URL (order: GPM_LLM_BASE_URL → stored value → OpenAI default). Local setups try several hosts until GET /v1/models succeeds. Saving clears the resolver cache. Test opens /api/llm-check in a new tab.

{% else %}

Einstellungen (/config), gespeichert in SQLite:

  • Scheduler-Intervall (Stunden): automatischer Scrape nach App-Neustart.
  • LLM-Anbieter: OpenAI-kompatibel oder Ollama (lokal, /v1).
  • Basis-URL, API-Schlüssel, Modell, Temperature.
  • Listings-Strategie (HTML → Kandidaten) und ein LLM-Prompt (Systemnachricht) für die JSON-Extraktion.

Die App löst die Basis-URL automatisch auf (GPM_LLM_BASE_URL → gespeicherter Wert → OpenAI-Standard). Beim Speichern wird der Resolver-Cache geleert. Test öffnet /api/llm-check.

{% endif %}

{{ _('docs.section.portals_llm') }}

{% if locale == 'es' %}

Portales (/portals): solo nombre, URL de búsqueda y Activo. En cada ejecución se visitan en secuencia (por ID) todos los portales activos. La estrategia de listado y los prompts están en Configuración (/config). Si el listado no incluye teléfono, el scraper intenta la ficha de detalle cuando el portal lo permite (p. ej. Idealista).

La primera base de datos vacía crea un portal semilla «Idealista Garraf». Los selectores HTML de los portales pueden romperse si el sitio cambia el diseño.

LLM: además de Config, puedes fijar GPM_LLM_BASE_URL en el entorno (p. ej. Docker/Kubernetes) para forzar la URL sin tocar la UI. Dentro de un contenedor, 127.0.0.1 apunta al propio contenedor: para Ollama en el host suele usarse http://host.docker.internal:11434 (ver README/compose).

{% elif locale == 'en' %}

Portals (/portals): only name, search URL, and Enabled. Each run visits enabled portals in sequence (by ID). Listing strategy and prompts live under Settings (/config). If the listing page omits a phone number, the scraper may open the listing detail page when the portal supports it (e.g. Idealista).

An empty database seeds a default «Idealista Garraf» portal. HTML selectors may need updates when a site changes layout.

LLM: besides Settings, set GPM_LLM_BASE_URL in the environment to force the base URL. Inside a container, 127.0.0.1 is the container itself—use http://host.docker.internal:11434 for host Ollama (see README/compose).

{% else %}

Portale (/portals): nur Name, Such-URL und Aktiv. Pro Lauf werden aktive Portale nacheinander (nach ID) verarbeitet. Strategie und Prompts stehen unter Einstellungen (/config). Fehlt die Telefonnummer auf der Übersicht, öffnet der Scraper bei unterstützten Portalen (z. B. Idealista) ggf. die Detailseite.

Leere DB: Standard-Portal «Idealista Garraf». Selektoren können bei Layout-Änderungen brechen.

LLM: optional GPM_LLM_BASE_URL in der Umgebung. Im Container ist 127.0.0.1 der Container selbst — Host-Ollama oft http://host.docker.internal:11434.

{% endif %}

{{ _('docs.section.runs') }}

{% if locale == 'es' %}

Ejecuciones (/runs): historial de scrapings con estado, recuentos de nuevos/actualizados y mensaje de error si aplica. Solo puede haber una ejecución a la vez; si intentas lanzar otra, verás un aviso. Puedes descargar el CSV de cada ejecución correcta que haya generado archivo.

El formulario de ejecución manual admite un ID de portal opcional para limitar el scrape a un portal concreto; vacío = todos los portales activos.

{% elif locale == 'en' %}

Runs (/runs): scrape history with status, new/updated counts, and error text when failed. Only one run at a time; a second trigger shows a busy message. Download per-run CSV when available.

The manual form accepts an optional portal ID; empty means all enabled portals.

{% else %}

Läufe (/runs): Verlauf mit Status, Zählern und Fehlertext. Nur ein Lauf gleichzeitig. CSV pro Lauf herunterladen, falls vorhanden.

Optionale Portal-ID im Formular; leer = alle aktiven Portale.

{% endif %}

{{ _('docs.section.blacklist') }}

{% if locale == 'es' %}

Patrones por URL canónica exacta o prefijo. Las coincidencias no se muestran en el listado principal y se omiten al fusionar resultados nuevos.

{% elif locale == 'en' %}

Patterns are exact canonical URLs or prefixes. Matches are hidden from the main table and skipped when merging new scrape results.

{% else %}

Muster: exakte kanonische URL oder Präfix. Treffer werden in der Hauptliste ausgeblendet und beim Merge übersprungen.

{% endif %}

{{ _('docs.section.env') }}

{% if locale == 'es' %}
  • GPM_DATA_DIR: directorio de app.db, CSV en runs/ y perfil persistente de Playwright.
  • GPM_LLM_BASE_URL (opcional): prioridad sobre la Base URL guardada en Config.

Otras variables de aplicación pueden definirse vía .env según app/config.py (prefijo GPM_).

{% elif locale == 'en' %}
  • GPM_DATA_DIR: directory for app.db, runs/*.csv, and the Playwright profile.
  • GPM_LLM_BASE_URL (optional): overrides the stored Settings base URL.

Additional app settings may be loaded from .env per app/config.py (GPM_ prefix).

{% else %}
  • GPM_DATA_DIR: Verzeichnis für app.db, runs/*.csv, Playwright-Profil.
  • GPM_LLM_BASE_URL (optional): überschreibt die gespeicherte Basis-URL.

Weitere Einstellungen aus .env gemäß app/config.py (Präfix GPM_).

{% endif %}

{{ _('docs.section.api') }}

{% if locale == 'es' %}
  • GET /api/llm-check — JSON con la URL resuelta e informe de intentos (misma lógica que el scraper).
  • POST /api/run — cuerpo JSON opcional {"portal_id": 1}; devuelve metadatos del ScrapeRun. Respuesta 409 si ya hay un scrape en curso.
{% elif locale == 'en' %}
  • GET /api/llm-check — JSON with resolved URL and attempt report (same logic as the scraper).
  • POST /api/run — optional JSON {"portal_id": 1}; returns run metadata. 409 if a scrape is already running.
{% else %}
  • GET /api/llm-check — JSON mit aufgelöster URL und Versuchsprotokoll.
  • POST /api/run — optionales JSON {"portal_id": 1}; Metadaten des Laufs. 409, wenn bereits ein Scrape läuft.
{% endif %}

{{ _('docs.section.docker') }}

{% if locale == 'es' %}

La imagen copia el código al construir: tras cambios en app/ o plantillas, reconstruye la imagen (docker compose up -d --build o equivalente). Un simple reinicio del contenedor no sustituye el código empaquetado.

Para que el contenedor alcance Ollama o un túnel en el anfitrión, revisa el README (red bridge, host.docker.internal, binding en 0.0.0.0).

{% elif locale == 'en' %}

The image copies code at build time: after changes under app/ or templates, rebuild (docker compose up -d --build). Restarting the container alone does not refresh copied files.

See the README for reaching host Ollama/tunnels (bridge network, host.docker.internal, bind 0.0.0.0).

{% else %}

Die Image enthält den Stand beim Build: nach Änderungen an app/ oder Templates neu bauen (docker compose up -d --build). Nur restart reicht nicht.

README: Zugriff auf Host-Ollama/Tunnel (Bridge, host.docker.internal, Bind 0.0.0.0).

{% endif %}
